# 🎉 欢迎使用众生谱 v2.0

## 📖 5 分钟快速上手

### 第一步：检查环境

双击运行 `check_env.cmd`，确保所有环境都已就绪。

如果提示缺少 vevo 环境，工具会询问是否自动创建。

### 第二步：启动服务

双击运行 `start.cmd`，会自动：
1. 激活 conda 环境 vevo
2. 启动后端服务（端口 8000）
3. 启动前端服务（端口 5173）

等待几秒钟，直到看到两个命令行窗口都显示"服务已启动"。

### 第三步：打开应用

在浏览器中访问：http://localhost:5173

你会看到三栏布局：
- **左栏**：卷宗管理
- **中栏**：演化控制台
- **右栏**：关系图谱投影

### 第四步：创建第一个卷宗

1. 在左栏顶部输入框输入小说名称，如"凡人修仙传"
2. 点击 "+" 按钮
3. 新卷宗会出现在列表中，并自动选中

### 第五步：配置 LLM 模型

1. 点击左栏的 "🔮 法阵管理" 按钮
2. 在弹出的窗口中填写：
   - **模型名称**：通义千问-Plus
   - **协议类型**：选择 bailian
   - **Model ID**：qwen-plus
   - **Base URL**：自动填充
   - **API Key**：粘贴你的 API Key
3. 勾选 "设为默认模型"
4. 点击 "🧪 测试连接"
5. 看到 "✅ 连接成功" 后，点击 "➕ 添加"
6. 关闭窗口

### 第六步：开始分析

1. 在中栏的文本框中粘贴小说片段（建议 500-2000 字）
2. 确认顶部显示 "当前法阵: 通义千问-Plus"
3. 点击 "🔥 开始炼化" 按钮
4. 等待进度条完成（会显示百分比）

### 第七步：查看结果

分析完成后，右栏会自动显示关系图谱：
- **圆圈**：代表人物
- **箭头**：代表关系
- **拖拽**：可以移动节点和画布
- **滚轮**：可以缩放视图

## 💡 使用技巧

### 提高准确率
- 输入完整的章节，包含上下文
- 使用高质量模型（qwen-max, gpt-4）
- 多次分析同一文本，系统会自动去重

### 降低成本
- 使用经济型模型（qwen-turbo）
- 避免重复分析相同内容
- 配置多个账号的 API Key 轮换使用

### 提升速度
- 使用快速模型（qwen-turbo）
- 减少单次输入的文本量
- 关闭不必要的浏览器标签页

## 🐛 遇到问题？

### 服务无法启动

**症状**：双击 start.cmd 后窗口闪退或报错

**解决**：
1. 运行 `check_env.cmd` 检查环境
2. 确保 conda 环境 vevo 已创建
3. 手动运行 `start_backend.cmd` 查看错误信息

### 模型测试失败

**症状**：点击"测试连接"后显示错误

**解决**：
1. 检查 API Key 是否正确（没有多余空格）
2. 检查 Base URL 是否可访问
3. 检查网络连接（可能需要代理）
4. 查看后端终端的详细错误信息

### 图谱不显示

**症状**：分析完成但右侧空白

**解决**：
1. 按 F12 打开浏览器控制台，查看错误
2. 刷新页面（Ctrl+Shift+R）
3. 检查是否有节点数据（查看数据库）
4. 尝试切换到其他卷宗再切换回来

### 分析无结果

**症状**：进度条完成但没有新数据

**解决**：
1. 检查是否选择了 LLM 模型
2. 查看后端日志，确认 API 调用状态
3. 尝试使用不同的模型
4. 确保输入文本包含人物信息
